git冲突----在合并之前提交过.md

发表于2022-06-23|更新于2022-11-01
|阅读量:

场景：小明：在本地更新了x.py文件，并提交到了git仓库，小红在之前也修改了 x.py,但是又无法撤回或者说忘记修改了x.py的哪个地方，此时git pull便会报错

亲测如下：

  1. 先git stash
    将自己工作区中修改的内容暂时缓存到git栈中，也可以说是备份在git栈中

  2. git pull
    把小明提交上去的代码拉下来

  3. git stash pop 将git栈中缓存的代码恢复到工作区

如果无法解决,放弃本地代码，直接覆盖

git reset –hard

git pull
