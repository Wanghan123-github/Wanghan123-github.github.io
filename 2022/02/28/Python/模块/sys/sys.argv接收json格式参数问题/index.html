sys.argv接收json格式参数问题

发表于2022-02-28|更新于2022-11-01
|阅读量:

亲测

场景：使用sys.argv接收参数时，在服务器下和容器下接收到的同一个json参数不一致

1.例如：

python ./fy4_main.py exe {"times":{"clp":"20210112005542"},"outpath":"/output/fy4","fy4_l2":{"clp":"/CMADAAS/DATA/FY4A/L2/AGRI/2021//20210112/Z_SATE_C_BAWX_20210112005542_P_FY4A-_AGRI--_N_REGC_1047E_L2-_CLP-_MULT_NOM_20210112004500_20210112004917_4000M_V0001.NC"},"shp":"/input/xibei.shp"}

在某些环境下，接收到exe后面的json参数后，会发现双引号被直接省略，导致无法转为json字符串

如下：

['times:{clp:20210112005542}', 'outpath:/output/fy4', 'fy4_l2:{clp:/CMADAAS/DATA/FY4A/L2/AGRI/2021//20210112/Z_SATE_C_BAWX_20210112005542_P_FY4A-_AGRI--_N_REGC_1047E_L2-_CLP-_MULT_NOM_20210112004500_20210112004917_4000M_V0001.NC}', 'shp:/input/xibei.shp']

当将上面的参数执行json.loads(上面json参数)时，可能会报错：“No JSON object could be decoded”

解决方式如下：

在执行命令的命令中，json参数的前后加一个单引号
python ./fy4_main.py exe '{"times":{"clp":"20210112005542"},"outpath":"/output/fy4","fy4_l2":{"clp":"/CMADAAS/DATA/FY4A/L2/AGRI/2021//20210112/Z_SATE_C_BAWX_20210112005542_P_FY4A-_AGRI--_N_REGC_1047E_L2-_CLP-_MULT_NOM_20210112004500_20210112004917_4000M_V0001.NC"},"shp":"/input/xibei.shp"}'

此时接收的json参数args为（两种）：

['\'{"times":{"clp":"20210112005542"},"outpath":"/output/fy4","fy4_l2":{"clp":"/CMADAAS/DATA/FY4A/L2/AGRI/2021//20210112/Z_SATE_C_BAWX_20210112005542_P_FY4A-_AGRI--_N_REGC_1047E_L2-_CLP-_MULT_NOM_20210112004500_20210112004917_4000M_V0001.NC"},"shp":"/input/xibei.shp"}\'']

['{"times":{"clp":"20210112005542"},"outpath":"/output/fy4","fy4_l2":{"clp":"/CMADAAS/DATA/FY4A/L2/AGRI/2021//20210112/Z_SATE_C_BAWX_20210112005542_P_FY4A-_AGRI--_N_REGC_1047E_L2-_CLP-_MULT_NOM_20210112004500_20210112004917_4000M_V0001.NC"},"shp":"/input/xibei.shp"}']

此时未防止服务器下和容器内接收到的参数不一致，所以转为json字符串更全面的办法是：

try:
    dicts = json.loads(args[0])
except:
    dicts = json.loads(args[0].split("'")[1])
print(dicts)
