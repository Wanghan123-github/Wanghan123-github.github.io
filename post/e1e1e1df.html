HTTP、HTTPS协议

发表于2023-07-26|更新于2023-07-27|Web
|阅读量:

前言

HTTP和HTTPS利用TCP/IP协议进行数据传输，TCP协议通过三次握手建立连接，提供可靠的通信

区别

安全性：

http是超文本传输协议，信息是明文传输，安全性差。https是SSL/TLS加密传输协议，安全性较好，但https协议需要申请ca证书用于身份验证，因此还需要一定的费用。

响应速度：

http响应速度快，因为http只需要tcp的三次握手连接，客户端和服务器需要交换 3 个包，而 HTTPS除了 TCP 的三个包，还要加上 ssl 握手需要的 9 个包，所以一共是 12 个包。

端口：

http使用80端口，https使用443端口。

握手次数：

另外https的握手过程比http多加了一步加密的过程，也就是https需要四次握手：

  1. 第一次握手：客户端发起https请求申请，带有支持的协议版本，如TSL1.0，一个客户端生成的随机数，支持的加密算法，支持的压缩算法。
  2. 第二次握手：服务器收到客户端的请求后，判断版本是否一致，如果不一致会关闭对话，然后服务器生成一个随机数，CA证书，确认使用的加密算法。
  3. 第三次握手：客户端收到服务器的返回后，验证证书的合法性，以及解密证书验证等，验证通过后，发送给服务器以下信息：客户端会用CA证书解密出来的服务器公钥对一个随机数进行加密，编码改变通知标识，客户端握手结束通知（包含第一次握手时所有内容的hash值，给服务器进行校验）。
  4. 第四次握手：服务器收到后，会返回编码改变通知，服务器握手结束通知（包含第二次握手时所有内容的hash值，给客户端进行校验）。

工作原理

http

  1. 客户端发起一个HTTP请求，包括请求方法（如GET、POST）、请求的URL、请求头等。
  2. 服务器接收到请求后，根据请求的URL和服务器上的资源来处理该请求，生成HTTP响应。
  3. 服务器将生成的HTTP响应发送回客户端，包括响应状态码、响应头和响应体等信息。
  4. 客户端接收到服务器的响应后，根据响应的内容进行处理，例如显示页面内容、下载文件等。

https

  1. 客户端发起一个HTTPS请求，与服务器建立TLS/SSL连接。
  2. 服务器将自己的数字证书发送给客户端，包含公钥、证书的颁发机构等信息。
  3. 客户端验证服务器证书的合法性，包括检查数字签名、证书是否过期、域名是否匹配等。
  4. 如果证书验证通过，客户端生成一个用服务器公钥加密的随机对称密钥，发送给服务器。
  5. 服务器使用私钥解密客户端发送的随机对称密钥。
  6. 客户端和服务器之间使用对称密钥加密和解密数据进行通信，保证数据传输的安全性。
  7. 当通信结束时，客户端和服务器可以协商关闭TLS/SSL连接。

使用场景

http

适用于一般的数据传输场景，包括文本、图片等不敏感信息的传输。

https

适用于对数据安全性要求较高的场景，例如网上支付、银行网站、电子邮件等敏感信息的传输。
