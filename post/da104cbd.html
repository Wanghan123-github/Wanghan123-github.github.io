使用Pycharm同步服务器的代码到本地

发表于2022-04-15|更新于2022-12-06|Python
|阅读量:

场景：将gitlab上的代码拉取到服务器下运行，但又想将代码拉取到本地开发，让本地更改完代码之后同步更新远程服务器上的代码（此时的服务器和本地都可以拉去代码）

思路：先git clone下来到本地，然后pycharm打开此文件夹，利用pycharm部署代码到服务器，在配置部署时先在服务器创建一个同名文件夹，然后在配置完之后，上传到此文件夹下，并配置有任何修改都会同步到服务器下

  1. 拉取指定分支的代码
    git clone -b 指定分支名 “项目链接”

  2. 用pycharm打开此文件夹，并在服务器创建一个同名的文件夹

  3. 依次点击Tools—-> Deployment —-> Configuration…

  4. 依次点击 “+” —-> SFTP

  5. 按照步骤填写以下内容，用于连接服务器的配置

  6. 配置本地与服务器之间的映射目录

  7. 按照步骤点击

  8. 配置本地代码与服务器代码实时同步

  9. 上传本地的代码到服务器

  10. 此时在本地修改完你的代码，Ctrl+s之后，代码就同步更新到服务器了

二、部署完发现无法自动更新

pycharm 连接远程库已ok，但是总是无法自动上传代码。并且在Tools-> Deployment -> Options 选项中出现Default server or group of server is not configured. Nothing would be uploaded automatically 这句话的意思就是，因为没有配置默认的连接服务器，所以无法自动上传代码【惊！！pycharm竟然这么蠢？？？明明已经连到指定的服务器了，你还跟我说需要指定？？】
