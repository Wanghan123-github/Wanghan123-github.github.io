git常用命令

发表于2022-04-14|更新于2022-12-06|git
|阅读量:

生成ssh

ssh-keygen -t rsa -C “xxxxx@xxxxx.com”（连续三次回车）

验证是否配置成功

ssh -T git@github.com

Hi haiwang! You’ve successfully authenticated, but GitHub does not provide shell access.

如果验证过程现实22端口连接失败，可以使用一下命令

ssh -T -p 443 git@ssh.github.com

查看远程分支和本地分支

git branch -a

查看远程分支

git branch -r

同步远程分支

git fetch
同步远程指定分支：git fetch origin 指定分支

注意：（也就是本地可能只有master分支，但是同步之后就可以将远程的所有分支都拉取到本地了）

拉取指定的远程分支并在本地创建对应的分支

git checkout -b 本地分支 origin/远程分支名
分支更新后继续同步：git pull origin 指定分支名

创建分支

git checkout -b 分支名

提交分支到线上

git add .
git commit -m “”
git push origin 新的分支名

合并

git merge master(合并master的分支到当前所在分支)

撤销合并

如果你觉得你合并后的状态是一团乱麻，想把当前的修改都放弃，你可以用下面的命令回到合并之前的状态：
git reset –hard HEAD

删除本地分支

注意：操作此命令时，不能在此分支下，否则报错

git branch -d 本地分支

强制删除本地分支

git branch -D 本地分支

删除远程分支

git push origin -d 远程分支

git add指定文件

git add 文件名

将git add. 和 git commit 合并提交

git commit -am “介绍”

同一文件反复修改时，反复提交

git commit –amend -m “介绍”

举例：

# a.py 内容:
print("hello world")

# 此时git add + git commit

# 之后又修改了a.py，

print("Hello World")

# 此时可以使用 git commit --amend -m "介绍" 覆盖修改的文件

查看git配置（用户名、密码、邮箱等）

git config –list

修改git用户名

git config –global user.name “hwang”

将当前修改放入缓存区（建议使用下面的）

git stash

将修改放入缓存区并赋予版本

git stash save ‘1111’

查看缓存区记录

git stash list

# 显示内容
stash@{0}: On wanghan: stash test

应用缓存（也就是将缓存区的内容恢复到工作区）

git stash apply

注意；默认删除最近的stash@{0}记录
这个应用方法不会删除缓存区的 stash@{0} 缓存记录

git stash apply stash@{0}

应用指定版本的缓存

应用缓存（方法二）

git stash pop

注意：删除的是缓存区的第一个，并且会删除缓存区的此条缓存记录

但如果使用 git stash pop 后有冲突时，如何撤销?

可以使用 git reset –hard ，即可撤销 git stash pop 操作，将当前分支状态恢复。

删除缓存区（删除指定版本的缓存）

git stash drop stash@{0}

如果误用drop，删错缓存，可以根据使用完drop命令后，返回的ID，

“19a…”就是ID，然后：git stash apply ID 即可恢复

删除缓存区（方法二）

git stash clear

如果误删缓存，恢复办法！！！

  1. git show 编号
注意：这个编号就是你drop之后返回的一长串编号

  1. git merge 编号
    （将删除的缓存合并到本地 也就是恢复到本地来）
