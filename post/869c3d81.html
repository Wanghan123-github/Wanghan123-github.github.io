基于column的连接

merge方法

参数：

  • pd.merge(df1,df2,how=”inner”)

    inner：基于列的数据进行==交集==合并（默认）
    outer：基于列的数据进行==并集==合并
    left ：基于左dataframe列的数据合并
    right：基于右dataframe列的数据合并

  • pd.merge(df1,df2,on=”数据的列名”)

    数据的列名（用来合并的列名）

  • pd.merge(df1,df2,left_on=”beta”,right_index=True,how=”inner”)

    left_on：基于左边DataFrame的列”beta”进行交集合并，也就是指定左边某一列作为合并的索引
    right_on：基于右边DataFrame的列”beta”进行交集合并
    left_index：基于左边DataFrame的index进行交集合并，也就是指定左边DataFrame的index作为合并的索引
    right_index：基于右边DataFrame的index进行交集合并

1.1 内连接（交集）

how=”inner”，取相同列的交集

取共同列”alpha”的数据，交集，进行连接

1.2 外连接（并集）

how = “outer”，取相同列的并集

1.3 左连接

how=”left”，以左边数据的列为主，列是相同列

1.4 右连接

how = “right”，以右边数据的列为主，列是相同列

2.1 多列连接

多列连接的处理逻辑和单列的大同小异，多列是通过找到多个相同列的数据，然后基于连接的方式来合并数据，比如：内连接，取多个相同列的数据，然后取交集合并

取相同列”alpha”、”beta”，交集，进行合并

基于index的连接

merge方法

根据参数配置，取左边数据的合并索引和右边数据的合并索引，进行连接方式的合并

left_on：取左边数据的某一列下的数据作为index，也就是作为合并的索引
right_index：直接取右边数据的index作为合并的索引

设置参数suffixes：可以修改连接时相同列的后缀名

1.2 将指定的列设置为索引

set_index：设置指定列为索引

set_index参数：
keys : 要设置为索引的列名（如有多个应放在一个列表里）
drop : 将设置为索引的列删除，默认为True
append : 是否将新的索引追加到原索引后（即是否保留原索引），默认为False
inplace : 是否在原DataFrame上修改，默认为False
verify_integrity : 是否检查索引有无重复，默认为False

这里建议使用merge合并，因为merge和join方法基本上都都能实现相同的功能

concat方法

series类型的拼接方法

pd.concat([df1,df2],axis=0,join=”outer”)
拼接函数，分为行拼接和列拼接，默认是行拼接（axis=0）。
内置拼接方法，分为内拼接和外拼接，默认为外拼接（join=”outer”）
拼接的对象是pandas的数据类型

参数：

  • pd.concat([df1,df2],axis=0)

0：行拼接（默认）
1：列拼接

  • pd.concat([df1,df2],join=”outer”)

outer：外拼接，==并集==（默认）
inner：内拼接，==交集==
